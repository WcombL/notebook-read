# Go、Docker

## Go

技术是否能发展起来:
- 有没有一个比较好的社区
- 有没有一个工业化的标准
- 有没有一个或多个杀手级应用

其它因素  
- 学习曲线是否低，上手快
- 有没有一个不错的提高开发效率的开放框架。如Java的Spring，C++的STL
- 是否有一个或多个巨型的技术公司作为后盾
- 有没有解决软件开发中的痛点

Go语言特性
- Go语言容易上手
- Go语言解决了并发程序和写底层应用开放效率的痛点
- Go语言有Google这个世界一流技术公司在后面
- Go语言的杀手级应用是Docker

## Docker

- Docker上手容易
- Docker解决了运维中的环境问题以及服务调度的痛点
- Docker的生态圈中有大公司在后面助力
- Docker产出了工业界标准OCI
- Docker的社区和生态已经出现像Java和Linux那样的态势
